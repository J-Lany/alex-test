# Задача 1. Реализовать класс `Alphabet`

## Цель
Сделать отдельный класс, который хранит алфавит проекта и умеет работать с ним на базовом уровне.

Это **первая маленькая задача проекта**, а не весь проект целиком.
Сейчас не нужно делать шифрование, расшифровку, файлы или меню.

---

## Почему начинаем именно с этого
Шифр Цезаря работает только тогда, когда заранее понятно:

- какие символы входят в алфавит;
- в каком порядке они расположены;
- как проверить, есть ли символ в алфавите;
- как найти индекс символа;
- как получить символ по индексу.

Без этого нельзя нормально сделать:

- шифрование;
- расшифровку;
- brute force;
- статистический анализ.

То есть `Alphabet` — это фундамент для всей остальной логики.

---

## Где создать класс

```text
src/main/java/com/owner/cryptanalyzer/alphabet/Alphabet.java
```

Пакет:

```java
package com.owner.cryptanalyzer.alphabet;
```

---

## Что нужно реализовать
Нужно создать класс:

```java
Alphabet
```

Внутри класса должны быть:

1. одна структура данных, в которой хранится алфавит;
2. 4 метода для работы с алфавитом.

---

## Как хранить алфавит
### Что допускается в рамках этой задачи
Для первой версии можно выбрать **один** из двух вариантов:

- `String`
- `char[]`

Нужно выбрать **один** вариант и использовать его в классе `Alphabet`.

### Рекомендация опытного разработчика
Для самой первой версии проще начать со `String`, потому что у него уже есть готовые методы:

- `length()`
- `indexOf()`
- `charAt()`

`char[]` тоже является хорошим вариантом, но потребует чуть больше ручной работы.

### Важно
Сейчас не нужно использовать:

- `List<Character>`
- `Set<Character>`
- `Map<Character, Integer>`

Это не запрет по ТЗ, а **ограничение текущей учебной задачи**, чтобы не усложнять старт.

---

## Какие символы должны входить в алфавит

### Что обязательно по ТЗ
Минимум, который должен быть в алфавите:

- русские буквы;
- пробел;
- знаки пунктуации:
  - `.`
  - `,`
  - `"`
  - `'`
  - `:`
  - `-`
  - `!`
  - `?`

В примерах к заданию также встречаются:

- `«`
- `»`

Их тоже можно включить в базовый вариант алфавита.

### Что можно добавить как улучшение
Это уже не обязательный минимум по ТЗ, а **осознанное улучшение**:

- добавить `ё`;
- добавить `Ё`;
- добавить заглавные буквы.

### Почему это разумное улучшение
- `ё` — часть русского алфавита;
- заглавные буквы часто встречаются в реальном тексте;
- такой алфавит будет лучше работать на обычных текстах, а не только на упрощённых примерах.

---

## Какие символы можно пока не включать
Это уже не требование ТЗ, а **рекомендация для первой версии**.

Можно пока не включать в алфавит:

- цифры;
- перевод строки;
- табуляцию;
- другие спецсимволы.

На этой задаче достаточно уметь определить, что символ **не входит** в алфавит.

Что именно делать с такими символами при шифровании — это **не часть этой задачи**.

---

## Методы, которые обязательно должны быть

### 1. Проверка наличия символа в алфавите
```java
public boolean contains(char ch)
```

Что делает:
- возвращает `true`, если символ есть в алфавите;
- возвращает `false`, если символа нет.

---

### 2. Поиск индекса символа
```java
public int indexOf(char ch)
```

Что делает:
- возвращает индекс символа, если он найден;
- возвращает `-1`, если символ не входит в алфавит.

---

### 3. Получение символа по индексу
```java
public char getCharByIndex(int index)
```

Что делает:
- возвращает символ, стоящий в алфавите на позиции `index`.

### Рекомендация для текущей версии
На этой задаче отдельную обработку неправильного индекса можно не делать.
Пока допустимо оставить стандартное поведение Java.

---

### 4. Получение размера алфавита
```java
public int size()
```

Что делает:
- возвращает количество символов в алфавите.

---

## Что НЕ нужно делать в этой задаче
Это не список "запрещено по ТЗ", а **границы текущей учебной задачи**.

Сейчас не нужно:

- писать шифрование;
- писать расшифровку;
- делать сдвиг символов;
- работать с ключом;
- читать файлы;
- писать в файлы;
- делать `main()`;
- использовать `Scanner`;
- писать brute force;
- писать статистический анализ.

Сейчас задача только одна: **создать класс, который описывает алфавит проекта и даёт к нему доступ через простые методы**.

---

## Ожидаемый результат
После выполнения задачи должен существовать класс `Alphabet`, который можно использовать так:

```java
Alphabet alphabet = new Alphabet();

alphabet.contains('а');
alphabet.contains('1');
alphabet.indexOf('б');
alphabet.getCharByIndex(0);
alphabet.size();
```

Если ты выбрала улучшенный вариант алфавита, то дополнительные проверки тоже должны работать, например:

```java
alphabet.contains('Ё');
alphabet.contains('А');
```

---

## Критерии готовности
Задача считается выполненной, если:

- создан класс `Alphabet`;
- в классе есть одна структура данных для хранения алфавита (`String` или `char[]`);
- в коде явно видно, какой состав алфавита выбран;
- в алфавите есть минимум, который требуется по ТЗ:
  - русские буквы;
  - пробел;
  - базовая пунктуация;
- реализованы методы:
  - `contains(char ch)`;
  - `indexOf(char ch)`;
  - `getCharByIndex(int index)`;
  - `size()`;
- `contains('а')` возвращает `true`;
- `contains('1')` возвращает `false`;
- `indexOf('1')` возвращает `-1`.

### Если выбран улучшенный вариант
Тогда дополнительно ожидается, что:

- `contains('Ё')` возвращает `true`;
- `contains('А')` возвращает `true`.

---

## Короткий итог
На этой задаче нужно не "делать шифр", а **подготовить фундамент для шифра**.

То есть:

- создать отдельный класс `Alphabet`;
- зафиксировать набор символов;
- отделить обязательный минимум по ТЗ от собственных улучшений;
- дать минимальный набор методов для работы с этим алфавитом.

Это и есть правильная первая маленькая задача для старта проекта.
